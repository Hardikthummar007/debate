Database Tables (Planned / JPA Entity Managed):
----------------------------------------------
- `users`: Stores registration, login info, and debate stats.
- `debate_session`: Stores topic, participants, active turns, and stances.
- `round`: Tracks scores and winner for each round (3 rounds total).
- `argument`: Stores user argument content, individual judge scores, reasoning, and final scores.
- `matchmaking_queue`: Temporary queue to match users by topic and opposite stance.

Important Decisions:
--------------------
- **Stateless AI evaluation reuse**: We are integrating with the existing python evaluate server (`server.py` on port 8000) by making HTTP requests from Spring Boot. This preserves `td.py` completely intact as requested.
- **Short Polling for Matchmaking & Turn-Sync**: Simple, robust polling of API status on the frontend to check if matchmaking succeeded or if the opponent has completed their turn.
- **Vanilla CSS Core styling**: Custom visually rich CSS variables and dark-mode styling implemented in Vanilla CSS for fine control and performance.
